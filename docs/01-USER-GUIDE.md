# SWTC 3D 列印系統 — 使用說明

> **版本**：v2.1（portal.html 統一入口）
> **更新日期**：2026/07/01
> **負責人**：jiliao

---

## 一、系統概覽

本系統是公司內部 3D 列印機（Formlabs Form4 / Form4L 等）的整合管理工具，**統一入口為 `portal/portal.html`**。

### 功能模組

| 模組 | 入口 | 說明 |
|------|------|------|
| **工作看板** | portal.html 內建 | 訂單進度追蹤 |
| **異常與資源** | portal.html 內建 | 異常紀錄、IPA / 設備維護 |
| **後台管理** | portal.html 內建 | 使用者管理、系統設定 |
| **3D列印機預約** | portal.html → iframe | 機台預約、甘特圖、即時狀態 |
| **材料庫存管理** | portal.html → iframe | 樹脂罐、備料庫存、消耗紀錄、月度分析 |

---

## 二、使用者角色

| 角色 | 預約系統 | 庫存系統 | 工作看板 / 異常 | 後台管理 |
|------|---------|---------|----------------|---------|
| **viewer** | 查看所有預約、機台狀態 | 查看庫存、消耗紀錄 | 查看 | — |
| **editor** | viewer ＋ 新增/編輯/刪除預約 | viewer ＋ 編輯庫存、新增入庫 | 新增/編輯 | — |
| **admin** | editor 全部 | editor ＋ 觸發同步、安全庫存設定、去重 | 全部 ＋ 刪除 | ✅ |

新使用者建立後預設為 viewer，admin 至 Firestore `users/{uid}` 修改 `role` 欄位，或透過後台管理頁面操作。

## 三、功能模組說明

### 3.1 工作看板

追蹤訂單從報價到出貨的各階段進度。

- **新增訂單**：填入訂單編號、客戶、機型、狀態
- **狀態欄**：可拖拉或點選切換（報價中 / 確認 / 生產中 / 出貨 / 完成）
- **排序**：可按各欄位排序

### 3.2 異常與資源

分三個子頁籤：

| 子頁 | 說明 |
|------|------|
| **異常紀錄** | 列印異常、設備故障等事件 |
| **IPA 耗材** | IPA 入庫 / 耗用紀錄 |
| **設備維護** | 定期保養、維修紀錄 |

### 3.3 後台管理（admin only）

- 使用者清單：查看所有帳號、角色
- 系統設定：workspace 全域設定

### 3.4 3D 列印機預約

#### 新增預約（editor / admin）

1. 點頂部「+ 新增預約」
2. 填入：日期、業務、機台、客戶/案號/備註、是否已下單、目的、負責工程師、狀態
3. 點「儲存」

#### 查看預約

- **甘特圖模式**（預設）：按機台分線顯示時間軸
- **表格模式**：列表，可排序、過濾

#### 即時機台狀態

頁面下方顯示機台即時狀態（IDLE / PRINTING / ERROR）、材料與剩餘量。Cloud Function 每 10 分鐘同步，前端 onSnapshot 即時推送，**不需手動重整**。

### 3.5 材料庫存管理

#### 四個分頁

| 分頁 | 內容 |
|------|------|
| **庫存總覽** | 月度消耗統計、低庫存警告、機台樹脂罐狀態 |
| **材料庫存** | 備料庫存清單、機台樹脂罐（摺疊）、可編輯 |
| **消耗紀錄** | 列印消耗、入庫、調整的歷史，預設 30 天 |
| **月度分析** | 圓餅圖、月度趨勢圖、詳細統計表 |

#### 材料庫存頁欄位說明

| 欄位 | 來源 | 可編輯 |
|------|------|-------|
| **材料名稱** | 系統內建 + 自動偵測新材料 | 否 |
| **庫存 (L)** | `inv.stock[material].total_ml / 1000` | ✓ editor 編輯模式 |
| **機台上 (L)** | Cloud Function 同步自 Formlabs API | 否（自動更新） |
| **安全庫存 (L)** | 手動設定 | ✓ admin only |
| **狀態** | 庫存 < 安全庫存 → 需叫料 | 否（自動） |
| **備註** | 手動輸入 | ✓ editor 編輯模式 |

#### 整頁編輯模式（editor / admin）

1. 點右上「✏️ 編輯」
2. 所有列的「庫存 (L)」和「備註」啟用（藍框）
3. 編輯完後點「✓ 儲存全部」→ 一次寫入 Firestore
4. 或點「✕ 取消」→ 還原

#### 新增入庫

點「📦 新增入庫」→ 選材料 + 數量 → 儲存。自動寫入一筆 history（type=stockin）。

#### 消耗紀錄

預設訂閱**最近 30 天**。點「⏮ 載入更早一個月」往前推 30 天（F5 後回預設）。

**紀錄類型：**

| Type | 顯示 | 說明 |
|------|------|------|
| `consume` | 🔴 列印消耗 | 列印完成 / 列印錯誤 |
| `aborted` | 🟠 列印中止 | 列印被取消 |
| `stockin` | 🟢 備料入庫 | 手動新增入庫 |
| `manual` | ⚙️ 手動調整 | 編輯庫存的差異紀錄 |

## 四、管理員（admin）專屬功能

### 庫存系統 admin 按鈕

| 按鈕 | 用途 |
|------|------|
| **🔄 觸發同步** | 立即執行 Cloud Function（一般每 10 分鐘自動） |
| **🗑 清除並重建** | 清空所有 history + last_processed_prints，重新 backfill |
| **🚫 去除重複** | 按 print_guid 去重，刪除多餘紀錄 |
| **⚙️ 安全庫存設定** | 設定每個材料低庫存警告閾值 |
| **🚫 隱藏材料設定** | 把舊版本材料從清單隱藏 |

## 五、典型使用情境

### 情境 A：列印機完成一筆列印

1. Formlabs 機台列印完成
2. **10 分鐘內** Cloud Function 自動偵測並寫入 Firestore
3. 前端 onSnapshot 即時更新：
   - 機台狀態卡片更新（IDLE）
   - 消耗紀錄出現新一筆
   - 庫存頁「機台上 (L)」更新

### 情境 B：使用者手動換樹脂罐

1. 使用者拿備料一罐裝到機台上
2. Cloud Function 同步後「機台上 (L)」更新為新罐容量
3. ⚠️ **備料庫存「庫存 (L)」不會自動扣** — 需手動編輯：
   - 材料庫存頁 → ✏️ 編輯 → 該材料庫存減少 1 L → 儲存

### 情境 C：admin 設定安全庫存

1. 材料庫存頁 → 點「⚙️ 安全庫存設定」
2. 設定每個材料閾值（L）→ 儲存
3. 庫存低於閾值的材料顯示「需叫料」紅色標籤

### 情境 D：消耗紀錄出現 1970 年資料

Cloud Function 內建 `parse_valid_ts` 修正：Formlabs 偶爾回傳 epoch(1970) 的 `print_finished_at` 時，自動退回 `created_at` 使用。正常情況下使用者不需要處理。

## 六、即時資料對應表

| 看到的數值 | 來源 | 更新頻率 |
|----------|------|---------|
| 預約清單 | Firestore `bookings` | 即時 (onSnapshot) |
| 工作看板 | Firestore `workboard_orders` | 即時 (onSnapshot) |
| 異常紀錄 | Firestore `issues_anomalies` | 即時 (onSnapshot) |
| 機台即時狀態 | Firestore `printer_status/current` | Cloud Function 10 分鐘寫入 → 即時推送 |
| 機台上樹脂罐剩餘 | Firestore `inventory/main.cartridges` | Cloud Function 10 分鐘寫入 → 即時推送 |
| 備料庫存 (L) | Firestore `inventory/main.stock` | 使用者手動編輯 → 即時 |
| 消耗紀錄 | Firestore `inventory_history`（最近 30 天訂閱） | Cloud Function 10 分鐘寫入 → 即時推送 |
